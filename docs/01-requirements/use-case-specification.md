## UC-02: Вход в систему

| Поле | Значение |
|------|----------|
| **Идентификатор** | UC-02 |
| **Название** | Вход в систему |
| **Акторы** | Anonymous (незарегистрированный пользователь) |
| **Предусловие** | Пользователь зарегистрирован в системе |
| **Постусловие** | Пользователь получает JWT-токен и получает доступ к защищённым ресурсам |
| **Триггер** | Пользователь открывает страницу входа и вводит учётные данные |

### Основной поток

| Шаг | Актор | Действие |
|-----|-------|---------|
| 1 | Пользователь | Открывает страницу входа (LoginPage / LoginView) |
| 2 | Пользователь | Вводит username и password |
| 3 | Пользователь | Нажимает «Войти» |
| 4 | Система (Control) | AuthController принимает POST /auth/login с LoginRequestDTO |
| 5 | Система (Mediator) | AuthService проверяет учётные данные через Spring Security |
| 6 | Система (Foundation) | UserRepository ищет пользователя по username |
| 7 | Система (Mediator) | AuthService генерирует JWT-токен с ролями пользователя |
| 8 | Система (Control) | AuthController возвращает LoginResponseDTO с токеном |
| 9 | Клиент | Сохраняет JWT в контексте (UserContext / AppState) |
| 10 | Система | Перенаправляет пользователя на главную страницу |

### Альтернативный поток A1: Неверные учётные данные

| Шаг | Действие |
|-----|---------|
| 5a | AuthService бросает BadCredentialsException |
| 6a | GlobalExceptionHandler возвращает HTTP 401 |
| 7a | Клиент отображает сообщение «Неверный логин или пароль» |

### Альтернативный поток A2: Вход через Google

| Шаг | Действие |
|-----|---------|
| 1a | Пользователь нажимает «Войти через Google» |
| 2a | Система перенаправляет на Google OAuth2 |
| 3a | Google возвращает код авторизации |
| 4a | OAuth2successHandler создаёт/обновляет пользователя и выдаёт JWT |

## UC-25: Создание задачи

| Поле | Значение |
|------|----------|
| **Идентификатор** | UC-25 |
| **Название** | Создание задачи на канбан-доске |
| **Акторы** | User (аутентифицированный участник проекта) |
| **Предусловие** | Пользователь является участником проекта; доска существует |
| **Постусловие** | Новая задача создана со статусом TODO и связана с доской |
| **Триггер** | Пользователь нажимает «Добавить задачу» на доске |

### Основной поток

| Шаг | Актор | Действие |
|-----|-------|---------|
| 1 | Пользователь | Открывает страницу проекта и выбирает доску (ProjectPage / ProjectBoardView) |
| 2 | Пользователь | Нажимает «Добавить задачу», вводит заголовок и описание |
| 3 | Пользователь | Подтверждает создание задачи |
| 4 | Система (Control) | TaskController принимает POST /api/projects/boards/{boardId}/tasks с TaskCreateDTO |
| 5 | Система (Control) | Проверяет валидность DTO (@Valid) |
| 6 | Система (Mediator) | TaskService устанавливает статус TODO, записывает createdAt |
| 7 | Система (Foundation) | KanbanTaskRepository сохраняет задачу в БД |
| 8 | Система (Mediator) | TaskService маппирует сущность в TaskResponseDTO |
| 9 | Система (Control) | Возвращает HTTP 200 с TaskResponseDTO |
| 10 | Клиент | Обновляет список задач на доске |

### Альтернативный поток A1: Пустой заголовок

| Шаг | Действие |
|-----|---------|
| 5a | @Valid выбрасывает MethodArgumentNotValidException |
| 6a | GlobalExceptionHandler возвращает HTTP 400 с описанием ошибки |
| 7a | Клиент отображает сообщение валидации |

### Альтернативный поток A2: Доска не найдена

| Шаг | Действие |
|-----|---------|
| 6a | TaskService бросает ResourceNotFoundException |
| 7a | GlobalExceptionHandler возвращает HTTP 404 |

## UC-09: Создание компании

| Поле | Значение |
|------|----------|
| **Идентификатор** | UC-09 |
| **Название** | Создание компании |
| **Акторы** | Admin (пользователь с ролью ADMIN) |
| **Предусловие** | Пользователь аутентифицирован; компании у него ещё нет |
| **Постусловие** | Компания создана и привязана к пользователю |

### Основной поток

| Шаг | Актор | Действие |
|-----|-------|---------|
| 1 | Admin | Переходит на страницу «Компания» |
| 2 | Admin | Вводит название компании и подтверждает |
| 3 | Система (Control) | CompanyController принимает POST /api/company с CompanyCreateDTO |
| 4 | Система (Mediator) | CompanyService создаёт Company, связывает с текущим пользователем |
| 5 | Система (Foundation) | Репозиторий сохраняет запись |
| 6 | Система | Возвращает CompanyResponseDTO с HTTP 200 |
