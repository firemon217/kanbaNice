# Расширенный глоссарий (системный уровень)

| № | Термин | Определение |
|---|--------|------------|
| 1 | **Kanban** | Методология визуального управления задачами, основана на досках и карточках |
| 2 | **Company** | Сущность-корень; объединяет пользователей и проекты под одной организацией |
| 3 | **KanbanProject** | Рабочая область внутри компании, включающая команду и доски |
| 4 | **KanbanBoard** | Набор задач, сгруппированных под единым названием внутри проекта |
| 5 | **KanbanTask** | Единица работы с заголовком, описанием и статусом (TODO/DONE) |
| 6 | **TaskStatus** | Перечисление состояний задачи: `TODO` — не начата, `DONE` — завершена |
| 7 | **ProjectMember** | Связующая сущность между User и KanbanProject, несущая роль участника |
| 8 | **ProjectMemberRole** | Роль участника проекта: `LEADER` (управляет) / `WORKER` (выполняет) |
| 9 | **RoleType** | Системная роль пользователя: `ADMIN`, `USER`, `AUDITOR` |
| 10 | **ADMIN** | Пользователь с полным доступом к управлению компанией и проектами |
| 11 | **USER** | Рядовой пользователь, работающий с задачами |
| 12 | **AUDITOR** | Пользователь с правом только на чтение данных |
| 13 | **AuthProviderType** | Способ аутентификации: `LOCAL` (email/пароль) или `GOOGLE` (OAuth2) |
| 14 | **JWT** | JSON Web Token — самодостаточный токен с подписью для аутентификации |
| 15 | **JWT-фильтр (JwtAuthFilter)** | Компонент Spring Security, извлекающий и проверяющий JWT из заголовка Authorization |
| 16 | **OAuth2** | Открытый протокол авторизации; используется для входа через Google |
| 17 | **AuthService** | Сервис аутентификации: реализует вход, регистрацию, генерацию JWT |
| 18 | **UserService** | Сервис управления профилем: обновление данных, смена пароля, удаление аккаунта |
| 19 | **CompanyService** | Сервис управления компанией и её сотрудниками |
| 20 | **ProjectService** | Сервис управления проектами и их участниками |
| 21 | **BoardService** | Сервис управления канбан-досками |
| 22 | **TaskService** | Сервис управления задачами: CRUD + маппинг в DTO |
| 23 | **DTO (Data Transfer Object)** | Объект передачи данных между слоями; не является сущностью JPA |
| 24 | **ModelMapper** | Библиотека для маппинга объектов (Entity ↔ DTO) |
| 25 | **Spring Data JPA** | Абстракция над JPA; автоматически реализует методы репозитория |
| 26 | **Repository** | Интерфейс слоя Foundation, реализующий доступ к данным в PostgreSQL |
| 27 | **GlobalExceptionHandler** | @ControllerAdvice-компонент, перехватывающий все исключения и формирующий HTTP-ответ |
| 28 | **ResourceNotFoundException** | Исключение, генерируемое при отсутствии запрошенной сущности (→ HTTP 404) |
| 29 | **BCrypt** | Алгоритм хеширования паролей с солью; применяется через PasswordEncoder Spring Security |
| 30 | **Swagger UI / SpringDoc OpenAPI** | Веб-интерфейс интерактивной документации REST API, доступный по /swagger-ui.html |
| 31 | **Docker** | Платформа контейнеризации; изолирует backend, frontend, nginx и PostgreSQL |
| 32 | **Docker Compose** | Инструмент оркестрации многоконтейнерного приложения через docker-compose.yml |
| 33 | **Nginx** | Высокопроизводительный веб-сервер / реверс-прокси; раздаёт статику и проксирует API |
| 34 | **CORS** | Cross-Origin Resource Sharing — политика контроля кросс-доменных запросов |
| 35 | **WebConfig** | Spring-компонент, настраивающий CORS-политику для REST API |
