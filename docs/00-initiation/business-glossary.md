# Бизнес-глоссарий

| № | Термин | Определение |
|---|--------|------------|
| 1 | **Kanban** | Методология визуального управления задачами, при которой работа представлена карточками, перемещаемыми между столбцами доски |
| 2 | **Компания (Company)** | Организационная единица в системе, объединяющая сотрудников под единым управлением |
| 3 | **Проект (Project)** | Целевая рабочая область внутри компании с набором участников и канбан-досок |
| 4 | **Канбан-доска (Board)** | Виртуальная доска, принадлежащая проекту, на которой размещаются задачи |
| 5 | **Задача (Task)** | Единица работы, размещённая на канбан-доске; имеет заголовок, описание и статус |
| 6 | **Статус задачи** | Состояние задачи: TODO (к выполнению) или DONE (выполнена) |
| 7 | **Участник проекта (ProjectMember)** | Сотрудник компании, включённый в проект с определённой ролью |
| 8 | **Роль в проекте** | Права доступа участника в рамках проекта: LEADER (руководитель) или WORKER (исполнитель) |
| 9 | **Роль в системе (RoleType)** | Уровень доступа пользователя в системе: ADMIN, USER или AUDITOR |
| 10 | **Администратор (ADMIN)** | Пользователь с полным доступом к управлению компанией, проектами и сотрудниками |
| 11 | **Пользователь (USER)** | Рядовой участник, управляющий своими задачами |
| 12 | **Аудитор (AUDITOR)** | Пользователь с правом только на чтение данных по задачам и проектам |
| 13 | **Руководитель проекта (Leader)** | Участник проекта, создающий доски и управляющий составом команды |
| 14 | **Исполнитель (Worker)** | Участник проекта, работающий с задачами |
| 15 | **Аутентификация** | Процесс подтверждения личности пользователя по логину и паролю или через OAuth2 |
| 16 | **JWT** | JSON Web Token — токен, используемый для безопасной передачи данных аутентификации |
| 17 | **OAuth2** | Протокол авторизации, позволяющий входить через сторонние сервисы (Google) |
| 18 | **REST API** | Архитектурный стиль взаимодействия клиента и сервера через HTTP с JSON-телами |
| 19 | **Сброс пароля** | Процесс восстановления доступа к аккаунту через одноразовый токен, присланный на email |
| 20 | **Контейнеризация** | Упаковка приложения и его зависимостей в Docker-контейнеры для воспроизводимого развёртывания |
