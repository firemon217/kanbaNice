# SWOT-анализ

## Анализ предметной области: управление проектами в небольших командах

| | **Положительные** | **Отрицательные** |
|--|---|---|
| **Внутренние** | **Сильные стороны (S)** | **Слабые стороны (W)** |
| | S1. Self-hosted решение — данные хранятся на собственном сервере | W1. Нет мобильного приложения (только web + desktop) |
| | S2. Два типа клиентов: веб и десктоп | W2. Статус задачи ограничен двумя состояниями (TODO/DONE) |
| | S3. JWT + OAuth2 (Google) аутентификация | W3. Отсутствие уведомлений в реальном времени (WebSocket не реализован) |
| | S4. Полная контейнеризация (Docker + Nginx) | W4. Нет тестового покрытия (0%) |
| | S5. REST API с документацией Swagger (27 эндпоинтов) | W5. Отсутствие пагинации в списках |
| | S6. Ролевая модель: системные роли + роли в проектах | W6. README изначально пустой |
| **Внешние** | **Возможности (O)** | **Угрозы (T)** |
| | O1. Добавление мобильного клиента (React Native / Flutter) | T1. Конкуренция с Trello, Jira, YouTrack |
| | O2. Расширение статусов задач (In Progress, Review, etc.) | T2. Уязвимости при неправильной настройке JWT |
| | O3. Интеграция WebSocket для real-time обновлений | T3. Потеря данных при отсутствии резервного копирования |
| | O4. CI/CD pipeline (GitHub Actions) | T4. Сложность администрирования Docker без DevOps-знаний |
| | O5. Публикация Docker-образа на Docker Hub | T5. Устаревание зависимостей (Spring Boot, React) |
| | O6. Добавление Prometheus + Grafana мониторинга | |

## Стратегические выводы

| Стратегия | Действия |
|-----------|----------|
| **S + O (использовать силу для возможностей)** | Использовать готовый REST API для добавления мобильного клиента; расширить Docker-инфраструктуру CI/CD |
| **W + O (устранить слабости через возможности)** | Добавить JUnit тесты; расширить статусы задач; реализовать WebSocket |
| **S + T (защититься силой)** | Усилить безопасность JWT: refresh-токены, rate limiting; настроить регулярные бэкапы PostgreSQL |
| **W + T (минимизировать риски)** | Добавить автоматизированное тестирование для предотвращения регрессий; покрыть документацией развёртывание |
