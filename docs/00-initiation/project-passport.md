## 2. БИЗНЕС-КОНТЕКСТ

**Проблема:**  
Небольшие и средние команды испытывают трудности с организацией рабочих процессов: задачи фиксируются в мессенджерах или таблицах, отсутствует единая точка управления проектами, трудно отследить ответственных и статус работ. Корпоративные решения (Jira, Trello) либо избыточны, либо требуют постоянного интернет-соединения.

**Решение:**  
Система kanbaNice предоставляет командам собственный развёртываемый сервис управления проектами в формате Kanban. Пользователи создают компанию, добавляют сотрудников, открывают проекты и распределяют задачи по доскам. Доступ возможен как через браузер, так и через нативное десктопное приложение.

---

## 3. ЦЕЛИ ПРОЕКТА

- Предоставить командам инструмент визуального управления задачами по методологии Kanban
- Обеспечить ролевую модель доступа (администраторы, участники проектов, аудиторы)
- Реализовать полнофункциональный REST API для интеграции с внешними системами
- Поддержать два типа клиентов: веб и десктоп — на единой серверной бизнес-логике
- Обеспечить безопасность данных: JWT-аутентификация, шифрование паролей, HTTPS
