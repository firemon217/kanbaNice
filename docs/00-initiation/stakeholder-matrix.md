# Матрица стейкхолдеров

| Стейкхолдер | Роль | Интерес в проекте | Влияние | Ожидания | Стратегия взаимодействия |
|-------------|------|-------------------|---------|----------|--------------------------|
| **Команда разработки** | Конечный пользователь (User/Worker) | Удобный инструмент отслеживания задач | Высокое | Простой UX, быстрый отклик, работа оффлайн (десктоп) | Регулярный сбор обратной связи, итеративная доработка UI |
| **Руководитель проекта** | Администратор проекта (Leader) | Управление задачами команды, видимость прогресса | Высокое | Полный контроль над досками и задачами | Приоритет реализации функций Leader |
| **Директор/Владелец компании** | Admin системы | Управление сотрудниками и проектами компании | Высокое | Безопасность данных, ролевая модель | Реализация ADMIN роли и политик доступа |
| **ИТ-администратор** | Администратор инфраструктуры | Развёртывание и поддержка системы | Среднее | Простота установки, Docker, документация | Руководство администратора, docker-compose |
| **Преподаватель** | Оценщик курсового проекта | Соответствие МУ, качество кода и документации | Высокое | Выполнение всех этапов, покрытие тестами | Подготовка полной документации по этапам |
| **Аудитор** | Наблюдатель (Auditor) | Прозрачность выполнения задач | Низкое | Просмотр задач без изменений | Реализация роли AUDITOR с read-only доступом |

## Анализ влияния / интереса

```
Влияние
  ^
В |  Директор       Преподаватель
ы |  Рук.проекта
с |  Команда разраб.
о |
к |                  ИТ-администратор
  |          Аудитор
Н +---+---+---+---+---+----> Интерес
    Н       С       В
```

**Высокий интерес + высокое влияние → управлять активно**  
**Низкий интерес + низкое влияние → наблюдать**
