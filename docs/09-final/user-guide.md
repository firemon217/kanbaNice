# РУКОВОДСТВО ПОЛЬЗОВАТЕЛЯ
## Система управления проектами kanbaNice

**Версия:** 1.0  
**Дата:** 25.06.2026

---

## 1. Введение

kanbaNice — корпоративная Kanban-система для управления задачами в командах. Данное руководство описывает порядок работы с системой через веб- или десктоп-клиент.

### Доступные клиенты
| Клиент | Установка | Особенности |
|--------|-----------|-------------|
| Web (React) | Не требуется (браузер) | OAuth2 Google, адаптивный дизайн |
| Desktop (JavaFX) | JRE 17 + JavaFX | Оффлайн-подготовка, нативный UI |

---

## 2. Регистрация и вход

### 2.1. Регистрация нового аккаунта
1. Откройте страницу регистрации (`/register` или экран Signup).
2. Заполните поля:
   - **Имя** — отображаемое имя в системе
   - **Логин** — уникальный идентификатор (латиница, цифры)
   - **Email** — для восстановления пароля и уведомлений
   - **Пароль** — минимум 6 символов
   - **Тип аккаунта**:
     - `LEADER` — может создавать компании и управлять сотрудниками
     - `WORKER` — исполнитель задач
3. Нажмите **«Зарегистрироваться»**.
4. После успешной регистрации вы будете автоматически авторизованы.

### 2.2. Вход в систему
- **По логину/паролю**: введите учётные данные на странице входа.
- **Через Google** (только Web): нажмите кнопку «Войти через Google» и подтвердите доступ.
- После входа JWT-токен сохраняется автоматически.

### 2.3. Восстановление пароля
1. На странице входа нажмите **«Забыли пароль?»**.
2. Введите email, указанный при регистрации.
3. Проверьте почту — придёт письмо со ссылкой для сброса.
4. Перейдите по ссылке и задайте новый пароль.

## 3. Работа с компанией

> ⚠️ Только пользователи с типом `LEADER` могут создавать компании.

### 3.1. Создание компании
1. Перейдите на страницу **«Компания»**.
2. Нажмите **«Создать компанию»**.
3. Введите название компании.
4. Нажмите **«Сохранить»**.

### 3.2. Управление сотрудниками
1. На странице компании перейдите во вкладку **«Сотрудники»**.
2. **«Добавить сотрудника»**: введите логин существующего пользователя → он будет привязан к вашей компании.
3. **«Удалить сотрудника»**: нажмите кнопку удаления напротив имени сотрудника.

---

## 4. Работа с проектами

### 4.1. Создание проекта
1. Перейдите на страницу **«Проекты»**.
2. Нажмите **«Новый проект»**.
3. Укажите название проекта.
4. Проект автоматически привязывается к вашей компании.

### 4.2. Добавление участников
1. Откройте карточку проекта.
2. В разделе **«Участники»** нажмите **«Добавить»**.
3. Выберите сотрудника компании и назначьте роль:
   - `LEADER` — управление проектом, досками, задачами
   - `WORKER` — выполнение задач, просмотр досок

### 4.3. Удаление проекта
Нажмите **«Удалить»** в карточке проекта. Подтвердите действие.  
⚠️ Удаляются все доски и задачи проекта безвозвратно.

---

## 5. Канбан-доски и задачи

### 5.1. Создание доски
1. Внутри проекта нажмите **«Новая доска»**.
2. Введите название доски (например, «Sprint 1», «Backlog»).

### 5.2. Создание задачи
1. На доске нажмите **«+ Добавить задачу»** в колонке TODO.
2. Заполните:
   - **Заголовок** (обязательно)
   - **Описание** (опционально)
3. Задача создаётся со статусом **TODO**.

### 5.3. Управление задачами
| Действие | Как выполнить |
|----------|---------------|
| Просмотреть задачу | Кликнуть по карточке задачи |
| Редактировать | Открыть задачу → изменить поля → «Сохранить» |
| Сменить статус | Перетащить карточку между колонками TODO ↔ DONE |
| Удалить задачу | Открыть задачу → «Удалить» |

### 5.4. Статусы задач
| Статус | Значение |
|--------|----------|
| TODO | Задача создана, но ещё не выполнена |
| DONE | Задача завершена |

## 6. Профиль пользователя

1. Нажмите на аватар/имя в верхнем меню → **«Профиль»**.
2. Доступные действия:
   - **Просмотр**: логин, email, тип аккаунта, компания
   - **Редактирование**: изменение отображаемого имени
   - **Удаление аккаунта**: необратимое удаление всех данных

## 7. Часто задаваемые вопросы

**Q: Я зарегистрировался как WORKER, но хочу создать компанию.**  
A: Создайте новый аккаунт с типом LEADER или попросите администратора изменить ваш тип (функция планируется).

**Q: Не приходит письмо для сброса пароля.**  
A: Проверьте папку «Спам». Убедитесь, что SMTP-сервер настроен корректно (см. Руководство администратора).

**Q: Можно ли работать одновременно в Web и Desktop?**  
A: Да, оба клиента используют единый REST API. Данные синхронизируются в реальном времени.

**Q: Что делать, если токен истёк?**  
A: Система автоматически перенаправит на страницу входа. Войдите повторно.
