### 1. ОБЩИЕ СВЕДЕНИЯ

#### 1.1. Наименование проекта
Корпоративная Kanban-система управления проектами и задачами «kanbaNice».

#### 1.2. Назначение системы
Автоматизация процессов планирования, распределения и отслеживания задач в командах малого и среднего размера по методологии Kanban. Обеспечение единого REST API для веб- и десктоп-клиентов с возможностью self-hosted развёртывания.

#### 1.3. Целевая аудитория
- Руководители проектов (Leader)
- Исполнители (Worker)
- Администраторы инфраструктуры
- Аудиторы (наблюдатели)

#### 2.3. Требования к архитектуре
Система должна быть построена по паттерну **PCMEF**:
- **Presentation**: React 19 (Web), JavaFX 21 (Desktop)
- **Control**: Spring MVC REST Controllers
- **Mediator**: Spring Services с бизнес-логикой
- **Entity**: JPA-сущности с бизнес-методами
- **Foundation**: Spring Data JPA Repositories

Зависимости направлены строго сверху вниз. Циклические зависимости запрещены.

#### 2.4. Требования к безопасности
- Хеширование паролей: BCrypt
- Аутентификация: JWT (JJWT 0.12.6) + OAuth2 (Google)
- Авторизация: системные роли (ADMIN, USER, AUDITOR) + проектные роли (LEADER, WORKER)
- Защита от SQL-инъекций: параметризованные запросы JPA
- Защита от XSS: экранирование на стороне React
- Транспортное шифрование: HTTPS через Nginx

### 3. ТРЕБОВАНИЯ К ИНТЕРФЕЙСАМ

#### 3.1. Веб-клиент (React)
- Минимум 5 страниц: Login, Register, Company, Projects, Board, Profile
- Адаптивная вёрстка
- Хранение JWT в localStorage
- Axios HTTP-клиент

#### 3.2. Десктоп-клиент (JavaFX)
- Минимум 5 экранов: Login, Signup, MainLayout, Projects, Board, Profile
- In-memory хранение JWT
- Собственный REST-клиент (`ApiClient`)

#### 3.3. REST API
- 27 эндпоинтов (полный список в Swagger UI)
- Версионирование: `/api/v1/` (подготовлено)
- Стандартные HTTP-статусы: 200, 201, 400, 401, 403, 404, 500
- Валидация входных данных через Bean Validation

### 4. ТРЕБОВАНИЯ К РАЗВЁРТЫВАНИЮ
